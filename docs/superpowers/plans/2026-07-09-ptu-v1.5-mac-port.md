## 已确认的 Mac 迁移阻塞点

### 1. `run.py --web` 曾经没有真正走 Web 模式

2026-07-09 已修复。

原症状：

```bash
python run.py --web
```

实际仍进入桌面模式，并调用 `desktop_app.py`。

同时 `_kill_port()` 使用 Windows 命令：

```text
netstat -ano | findstr :8000
taskkill /F /PID ...
```

在 macOS 上输出：

```text
findstr: command not found
netstat: illegal option -- o
```

当前状态：

```text
run.py --web 进入 FastAPI Web 模式
macOS 端口清理使用 lsof + SIGTERM
GET / -> 200
GET /api/build-id -> 200
GET /api/tasks -> 200
```

### 2. Mac 桌面壳可加载，但 pywebview 菜单结构崩溃

2026-07-09 已修复：macOS 下不再向 pywebview Cocoa 传入 Windows tuple 菜单。

原症状：执行 `python run.py` 后，pywebview Cocoa 报错：

```text
AttributeError: 'tuple' object has no attribute 'title'
```

触发位置在 `desktop_app.py` 调用 `webview.start(menu=...)` 后，macOS pywebview 6 的 Cocoa 实现不接受当前 Windows 版菜单 tuple 结构。

当前状态：

```text
python run.py 可启动 Mac 桌面窗口
banner: 1.5.0 - Mac 桌面客户端
webview.start 在 macOS 下不传 menu
```

### 3. Playwright 浏览器检测是 Windows 路径和 `.exe` 名称

2026-07-09 已修复。

实际安装位置：

```text
~/Library/Caches/ms-playwright/.../chrome-headless-shell-mac-arm64/chrome-headless-shell
```

项目检测函数：

```python
setup_check.check_playwright()
```

当前返回：

```text
True
```

原问题是 `setup_check.get_chromium_path()` 只搜索：

```text
~/AppData/Local/ms-playwright
~/.playwright
chrome-headless-shell.exe
headless_shell.exe
chromium-headless-shell.exe
chrome.exe
chromium.exe
```

当前状态：

```text
macOS 搜索 ~/Library/Caches/ms-playwright
支持 chrome-headless-shell 无 .exe 文件名
支持 chrome-headless-shell-mac-arm64
```

### 4. FFmpeg 配置优先命中 Windows 二进制

2026-07-09 已修复。

当前 `settings.ffmpeg_path` 返回：

```text
可执行的 macOS ffmpeg 路径
```

原问题是 macOS 可能命中仓库内 Windows FFmpeg：

```text
PE32+ executable (console) x86-64, for MS Windows
```

在 macOS 上不能执行。

当前本机可用 macOS FFmpeg 路径：

```text
可执行的 macOS ffmpeg 路径
可执行的 macOS ffprobe 路径
```

### 5. 右侧抖音预览是 Windows WebView2 / WinForms 实现

2026-07-09 已完成第一阶段降级：非 Windows 平台不会尝试加载 WebView2 / WinForms，JS API 返回结构化 `unsupported`，提示 Mac V1.5 版使用系统浏览器复制链接。

文件：

```text
backend/app/desktop_douyin_panel.py
```

当前实现依赖：

```text
Microsoft.Web.WebView2.WinForms
WebView2
WinForms
```

Mac 版不能复用这块实现。V1.5 Mac 版第一阶段应先降级为外部浏览器或普通 pywebview 页面，不把 WebView2 迁移作为第一天目标。

### 6. Mac 前端交互与 Windows 鼠标逻辑不同

2026-07-09 已完成第一轮前端迁移处理。

已处理：

- 标题栏拖拽从模板内联 `onmousedown` 改为 JS 初始化时绑定 `pointerdown`。
- 忽略非左键和真实触摸 `pointerdown`，避免触控/辅助点击误触拖拽。
- 剪贴板失败提示按平台显示，Mac 显示 `⌘V`，Windows 显示 `Ctrl+V`。
- Mac 桌面端收到右侧内嵌抖音预览 `unsupported` 后，不再显示“打开失败”，改为系统浏览器流程。
- Web UI 布局冒烟覆盖 1280 宽和 900 宽：
  - 1280 宽：右侧 dock 显示，无横向溢出。
  - 900 宽：右侧 dock 自动隐藏，无横向溢出。

## 2026-07-09 发布前审查结果

本轮审查范围：

```text
Mac 迁移 diff
Windows build.spec / Mac build_macos.spec 对比
Windows-only 依赖扫描：WebView2、WinForms、win32、explorer、ffmpeg.exe、ffprobe.exe
运行时路径：PTU_RUNTIME_DIR、Application Support、LOCALAPPDATA
封包态 FFmpeg / FFprobe / Playwright
前端鼠标、触控板、粘贴和外部浏览器降级
诊断包与敏感信息脱敏
```

已修复的逻辑问题：

```text
1. Mac 带音乐合成视频可能找不到包内 ffprobe
   根因：media_processor._probe_duration() 只找同级 ffprobe.exe，再回退到裸 ffprobe
   修复：按平台查找同级 ffprobe / ffprobe.exe
   回归测试：tests/test_media_processor_douyin_clean.py::test_probe_duration_uses_sibling_ffprobe_without_exe_on_macos

2. QR 登录 Playwright 回退路径仍默认 Windows AppData
   根因：qr_login._get_qrcode_pw() 写死 ~/AppData/Local/ms-playwright
   修复：新增 _default_playwright_browsers_path()，Mac 使用 ~/Library/Caches/ms-playwright
   回归测试：tests/test_login_status_feedback.py::test_qr_playwright_default_browser_path_is_platform_specific

3. 非 Windows 端口清理范围过宽
   根因：run._kill_port() 用 lsof 查询 tcp:{port}，未限制 LISTEN
   修复：增加 -sTCP:LISTEN，只终止监听该端口的进程
   回归测试：tests/test_run_setup.py::test_kill_port_does_not_use_windows_commands_on_macos
```

Windows / Mac 差异结论：

```text
Windows 版保留：WebView2 / WinForms 内嵌抖音 Dock、Inno Setup、Ptu_Setup_v1.5.0.exe、ffmpeg.exe / ffprobe.exe
```
