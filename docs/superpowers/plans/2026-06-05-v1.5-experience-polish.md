# Ptu v1.5 Experience Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make v1.5 feel like one coherent desktop workbench: left-side capture controls, right-side embedded Douyin browsing/login/tutorial, clearer progress, sharper live-photo output, and no flashing command windows.

**Architecture:** Keep FastAPI + pywebview as the main app shell, but replace the fake second Douyin window with a Windows-only native WebView2 child control mounted inside the main pywebview WinForms form. The HTML right dock owns layout and instructions; the Python bridge owns the child WebView2 lifecycle, navigation, cookie sync, and new-window interception.

**Tech Stack:** Python 3.13, FastAPI, pywebview WinForms/WebView2 backend, WebView2 .NET interop via pythonnet, vanilla JS/CSS, FFmpeg.

### 2026-06-10 二测修复记录

- 用户回传 `日志.zip` 后，日志显示 v1.5.0 安装版已使用内置 Chromium 和安装目录内置 FFmpeg；二维码 API 直调返回 403 后切换 Playwright 获取二维码成功，属于已有兜底路径。
- 日志中出现 `实况合成失败 live_0000 ... Conversion failed!`。根因在下载阶段的备用实况合成命令：视频流和末尾静态图 concat 前缺少统一时基、帧率、画布和像素格式，且失败日志只截最后三行，丢失 FFmpeg 有效错误。已在 `backend/app/services/downloader.py` 中统一为 `1080x1920`、`fps=30`、`settb=AVTB`、`yuv420p`，并保留包含 error/failed/invalid/do not match/conversion failed 的日志行。
- 用户反馈主窗口大小变化时右侧抖音窗口不跟随。根因是右侧 WebView2 子控件只依赖前端 `ResizeObserver` 推坐标，没有在原生层保存最后坐标并绑定 WinForms 主窗口 `Resize`、`SizeChanged`、`Move`。已在 `NativeDouyinPanel` 中保存 `_last_rect` 并在主窗口布局变化时重放边界；前端 `mountBrowserDock()` 增加 `window.resize`、`visualViewport.resize` 和持续同步计时器。
- 本轮新增测试覆盖：`tests/test_douyin_browser_panel.py` 检查右侧 Dock 同步链路，`tests/test_downloader_text_content.py` 检查实况合成 FFmpeg 参数和错误日志。
